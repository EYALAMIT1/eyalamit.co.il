## ⚠️ הודעת חשיבות

מסמך זה מגדיר את אופן הפעולה בפרויקט. **סטייה מנהלים אלו עלולה לגרום לעיכובים בפריסה.**

כל חבר צוות חייב לקרוא ולהבין את הנהלים הללו לפני תחילת העבודה.

---

## 👑 סמכות עליונה - CEO Authority (SSOT v4.7)

**עילאי אמית (CEO) הוא מקבל ההחלטות הסופי לכל אבני הדרך בפרויקט.**

**שרשרת הסמכות:**
1. **CEO (עילאי אמית)** - סמכות עליונה (רמה 5)
2. **ארכיטקט (צוות 0)** - תכנון פרויקט (רמה 4)
3. **Gatekeeper (צוות 3)** - אכיפת נהלים (רמה 4)
4. **QA (צוות 2)** - בדיקות איכות (רמה 3)
5. **פיתוח (צוות 1)** - יישום קוד (רמה 2)

---

## 👥 מבנה הצוותים (Team Structure)

הפרויקט מתנהל במודל שלושה צוותים נפרדים ב-Cursor:

### צוות 1 (פיתוח - Development)
- **אחריות:** יישום קוד ופיתוח טכני
- **פלט:** קוד מוכן לבדיקה

### צוות 2 (QA ומוניטור - QA & Monitoring)
- **אחריות:** בדיקה ואישור איכות
- **פלט:** אישור "Green Light" למעבר לפרודקשן

### צוות 3 (דוקומנטציה וגיט - Docs & Git)
- **אחריות:** ניהול Repository, אכיפת נהלים, דוקומנטציה
- **תפקיד:** Gatekeeper - שומר הסף לכל המעברים

## 📝 פורטוקול דיווח מחייב (Mandatory Reporting Protocol)

**⚠️ חובה! כל הודעה בין הצוותים חייבת לעמוד במבנה הבא - העתק והשתמש בבלוק הקוד:**

```
From: [שם הצוות]
To: [שם הצוות המקבל]
Subject: [נושא ההודעה]
Status: [PENDING/IN_PROGRESS/COMPLETED/FAILED/BLOCKED]
Done: [פירוט טכני של מה שבוצע]
Evidence: [נתיב לקובץ/לוג/דוח]
Blockers: [גורמים מעכבים או 'None']
Next: [צעד הבא מיידי]
Timestamp: [YYYY-MM-DD HH:MM]
Extra details in professional report: [YES/NO]
```

**חובה להשתמש בפורמט זה בכל תקשורת בין הצוותים.**

## 1. ניהול גרסאות (Git)

### ענף בסיס
- **ענף בסיס:** `production-current` (גרסה נקייה מהפרודקשן)
- **איסור:** אין לבצע שינויים ישירים בענף זה

### Branching Strategy
- **פורמט:** `task-[id]-[desc]`
- **דוגמאות:**
  - `task-001-fix-seo-issues`
  - `task-002-optimize-images`
  - `task-003-security-hardening`

### Commits
- **שפה:** אנגלית בלבד
- **פורמט:** `[type]: [description]`
- **דוגמאות:**
  - `fix: update wp-core to v6.8.3`
  - `feat: add image optimization`
  - `refactor: optimize database queries`

### Merge Strategy
- **איסור:** אין לבצע Merge ל-Main ללא אישור מפורש מצוות 3 (Gatekeeper)
- **תהליך:** Code Review → אישור צוות 2 (QA) → אישור צוות 3 → Merge

---

## 2. סביבת פיתוח

### סביבה מותרת
- **עבודה אך ורק בתוך Docker**
- **איסור:** אין לבצע שינויים ישירים בשרת הפרודקשן או ב-Staging

### סנכרון
- **חובה:** כל שינוי חייב סנכרון לאחור ל-Git
- **כלי:** Docker Compose + Git

### Debug ו-Logs
- **חובה:** כל שגיאה ב-`debug.log` חייבת לקבל התייחסות מיידית
- **פעולה:** תיקון/הסבר/דוח לכל שגיאה

---

## 3. בקרת איכות (QA)

### דוח ביצוע חובה
בסיום כל משימה, יש לשלוח דוח ביצוע הכולל:

#### רכיבי הדוח
- **פלטי טרמינל** - כל הפקודות שהורצו
- **לוגים** - debug.log ו-logs רלוונטיים
- **צילומי מסך** - של הבדיקה המקומית
- **תוצאות בדיקות** - פונקציונליות ו-performance

#### תהליך ביקורת
1. משימה הושלמה → דוח נשלח לצוות 2
2. צוות 2 בודק ומאשר
3. צוות 3 מבצע Code Review ואכיפת נהלים
4. אישור → מעבר לשלב הבא
