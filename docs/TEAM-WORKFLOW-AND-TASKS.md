# 📋 הנחיות עבודה וחלוקת משימות

## מבנה הצוותים וזרימת העבודה

### צוות 1 (פיתוח) → צוות 2 (QA) → צוות 3 (Gatekeeper) → פרודקשן

---

## תחומי אחריות מפורטים

### צוות 1: פיתוח
**אחריות:** יישום טכני של כל השינויים
- פיתוח ויישום קוד
- בדיקות בסיסיות מקומיות
- דיווח על התקדמות לפי הפרוטוקול

### צוות 2: QA ומוניטור
**אחריות:** בדיקה ואישור איכות
- בדיקות פונקציונליות
- בדיקות ביצועים
- בדיקות אבטחה
- מתן "Green Light" לצוות 3

### צוות 3: דוקומנטציה וגיט (Gatekeeper)
**אחריות:** שמירת סדר וניהול תנועה
- ניהול Repository ו-Branches
- אכיפת נהלי SOP
- אישור סופי לפני Merge ל-Main
- עדכון דוקומנטציה
- דיווח לארכיטקט (Team 0)

---

## תהליך עבודה סטנדרטי

1. **צוות 1** מקבל משימה ומפתח
2. **צוות 1** שולח דיווח COMPLETED לצוות 2
3. **צוות 2** מבצע QA ומעניק Green Light
4. **צוות 2** שולח דיווח לצוות 3
5. **צוות 3** מבצע Code Review ואכיפת נהלים
6. **צוות 3** מאשר Merge ל-Main
7. **צוות 3** מדווח לארכיטקט (Team 0)

## כללי משחק

- **אין מעבר שלב ללא אישור הצוות הקודם**
- **צוות 3 הוא ה-Gatekeeper הסופי**
- **כל דיווח חייב להיות בפורמט המחייב**
- **סטייה מהנהלים = חסימה על ידי צוות 3**
